### 1. Цель работы
Начать практическую реализацию проекта, освоить работу с ветвлением в Git (Feature Branch Workflow) и применить конвенцию коммитов.

---

### 2. Ход работы

#### 2.1. Подготовка к разработке
Перед началом работы была актуализирована ветка `main`.
Для выполнения текущих задач из бэклога была создана отдельная ветка:
`feature/core-setup`

#### 2.2. Настройка окружения
1. Создано виртуальное окружение `venv`.
2. Установлены основные зависимости:
   - `aiogram` (для работы с Telegram API)
   - `python-dotenv` (для безопасного хранения токенов)
3. Создан файл `.gitignore`, исключающий загрузку системных файлов и секретов (`.env`) в репозиторий.
4. Сгенерирован файл `requirements.txt`.

#### 2.3. Реализация базового функционала
Написан код "ядра" бота:
- **`main.py`**: Точка входа. Настраивает логирование, инициализирует бота и запускает процесс Polling (опрос сервера Telegram).
- **`handlers.py`**: Модуль с обработчиками сообщений. Реализована асинхронная функция `cmd_start`, обрабатывающая команду `/start`.

#### 2.4. Работа с Git и GitHub
В процессе работы использовались **Conventional Commits**:
- `feat: add basic project structure and dependencies`
- `feat: implement start command handler`

После завершения разработки ветка была отправлена в удаленный репозиторий. Оформлен **Pull Request**, который затем был успешно слит (Merged) в ветку `main`.
