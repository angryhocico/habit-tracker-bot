### 2. Описание проекта
Разрабатываемый продукт — **Habit Tracker Bot**.
Это Telegram-бот, который помогает пользователям формировать полезные привычки.
**Основные функции (MVP):**
- Регистрация пользователя.
- Создание новой привычки (название, описание).
- Ежедневные напоминания и отметка выполнения.
- Просмотр простой статистики (сколько дней подряд выполнено).

**Выбранный стек технологий:**
- **Язык:** Python 3.10+
- **Фреймворк:** aiogram 3.x (асинхронная работа с Telegram API)
- **База данных:** SQLite (на этапе разработки), PostgreSQL (в продакшене)
- **Управление проектом:** GitHub Projects & Issues
- **Документация/Лендинг:** GitHub Pages

### 3. Ход работы

#### 3.1. Создание репозитория
Был создан публичный репозиторий на GitHub. Настроены файлы `.gitignore` (для Python) и лицензия.
Репозиторий клонирован на локальную машину для дальнейшей разработки.

#### 3.2. Оформление документации (README.md)
В корне проекта создан файл `README.md`, содержащий:
- Название и описание проекта.
- Цели и задачи.
- Описание используемого стека.
- Инструкцию по запуску (заготовка).

#### 3.3. Организация процесса разработки (GitHub Projects)
Создана доска проекта (Kanban-board) во вкладке **Projects**.
Настроены статусы задач:
1. **Todo** (Запланировано)
2. **In Progress** (В работе)
3. **Done** (Готово)

#### 3.4. Формирование бэклога (Issues)
Проведен мозговой штурм и декомпозиция задач. Созданы Issues (задачи) для первых этапов разработки:
1. **[Core]** Инициализация проекта и настройка окружения (poetry/venv, .env).
2. **[Bot]** Регистрация бота в @BotFather, получение токена.
3. **[Bot]** Реализация базового обработчика команды `/start`.
4. **[DB]** Проектирование схемы БД (таблицы `users`, `habits`).
5. **[Feature]** Реализация FSM (машины состояний) для добавления привычки.
6. **[Docs]** Создание страницы-визитки на GitHub Pages.

#### 3.5. Реализация GitHub Pages
