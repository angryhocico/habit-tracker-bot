# Отчет по лабораторной работе №3
## Тема: Интеграция кода (First Integration)

**Выполнил:** Есин И.Д.
**Тема проекта:** Телеграм-бот "Habit Tracker"

---

### 1. Цель работы
Научиться работать с Pull Request (PR), проводить интеграцию нового функционала в основную ветку (`main`) и имитировать процесс Code Review.

---

### 2. Ход работы

#### 2.1. Постановка задачи
Необходимо было внедрить слой хранения данных.
Задача из бэклога: **[DB] Проектирование схемы БД**.
Для решения выбрана СУБД **SQLite** (как наиболее простая для MVP).

#### 2.2. Разработка в ветке
Создана ветка `feature/database-init`.
Реализован модуль `database.py`:
- Функция `init_db()`: создает таблицы `users` и `habits` (если их нет).
- Функция `add_user()`: добавляет Telegram ID и username пользователя в базу, игнорируя дубликаты.
- В `main.py` добавлена инициализация БД при запуске.
- В `handlers.py` добавлен вызов регистрации пользователя при команде `/start`.

#### 2.3. Создание Pull Request
После завершения разработки изменения были отправлены на GitHub.
Оформлен Pull Request с описанием проделанной работы и инструкцией для тестирования.

#### 2.4. Code Review и Merge
Проведен процесс самопроверки (Code Review) через интерфейс GitHub.
Были проверены изменения в файлах, оставлены примечания.
После успешной проверки PR был слит (Merged) в ветку `main`.
Конфликтов слияния не возникло, так как работа велась в новых файлах.
